Tak

(Build)

      Tak was originally a fictional board game imagined by Patrick Rothfuss. This board game is similar to tic tac toe, except with a few modifications that make the game feel almost similar to chess. Once I had a playable version of the game running I could start working on an AI to play against.

      For the first attempt I implemented Alpha Beta Pruning, which has pros and cons. The implementation was sound, however the concept of MiniMaxing as a whole is not optimal for a game like Tak. It has countless board states especially as the stones(game pieces) stack higher. When MiniMaxing generates the full breadth of future boards the “right” heuristic will find the “right” move. This works for small scale things and struggles here. No matter how much optimization you throw at the problem, given the number of legal moves allowed, searching beyond 3 or 4 moves with a game like this is infeasible for this algorithm.

      After this experience I worked with a friend on some more well-trodden ground, a chess algorithm.

Chess Investigation

      We investigated several different search algorithms and to implement them effectively. After some deliberation we decided to implement the Monte Carlo Tree Search. This algorithm is particularly good with the sorts of constraints games like chess and tak deal with, and resembles how I think about problem solving. Try to analyze what the “best” move is and don't think about the rest. This algorithm's strength is also its greatest pitfall. Similar to humans, it can’t think of everything; being based on whatever random moves it analyzes using a heuristic to value check the boards produced.

      While this algorithm is not perfect it is extremely effective, securing fourth place in the class chess tournament! I also particularly like it, not despite, but because of its human flair. With this investigation and contest aside we can talk about next steps.

Comic Book Filter

      Using these encapsulated, delicate tools, with the larger skills of game programming that I have mastered I can create customized effects that are especially fascinating. During this project I was once again working in a team, using procedural generation with post processing effects to stylize randomized scenes. I implemented the Kuwahara Filter to tailor the look of our project.

A Brief History

      The Kuwahara filter, known for its painterly effect, was originally invented in Japan as heart image denoising software in the 1970s. This is why it creates the blocks of color, resembling brush strokes.

Updates!

      Building on this foundation using a few other techniques we can tweak the look! Some blur here, a depth based outline filter there, and presto we can update this filter from the 70s to get a new effect! This stylistically aligns with my vision, but it is not optimized. In its current state the filter is an interesting exercise but not universally supported. If I were to actually use this in a game I would need to focus on optimizing these effects.
